## 1. Research Synthesis: Real-World Datasets

Three complementary datasets have been selected to cover all three modalities of the FraudLens pipeline.

### 1.1 Tabular / Metadata — IEEE-CIS Fraud Detection

| Property | Value |
|---|---|
| **Source** | Kaggle — `ieee-fraud-detection` |
| **Records** | ~590,540 transactions |
| **Features** | 392 (TransactionAmt, V1–V339, C1–C14, D1–D15, M1–M9, identity cols) |
| **Fraud Rate** | ~3.5% (highly imbalanced) |
| **Labels** | `isFraud` binary |
| **Download** | `python -m src.data.download_data` (requires `KAGGLE_USERNAME` / `KAGGLE_KEY`) |

**Why chosen:** Gold standard for academic fraud detection benchmarks. Contains rich temporal, device, and behavioral features that mirror production card-not-present fraud scenarios.

### 1.2 Tabular / Alternate — PaySim (Mobile Money)

| Property | Value |
|---|---|
| **Source** | Kaggle — `ealaxi/paysim1` |
| **Records** | ~6.3 M synthetic mobile money transactions |
| **Features** | 11 (step, type, amount, nameOrig, oldbalanceOrg, newbalanceOrig, …) |
| **Fraud Rate** | ~0.13% |
| **Labels** | `isFraud`, `isFlaggedFraud` |
| **Download** | Place `paysim.csv` at `data/paysim/paysim.csv` |

**Why chosen:** Provides a complementary transaction distribution (African mobile banking) that improves generalization when mixed with IEEE-CIS.

### 1.3 Images — SSBI Check Forgery Dataset

| Property | Value |
|---|---|
| **Source** | GitHub — `saifkhichi96/ssbi-dataset` |
| **Images** | ~800 real bank check scans |
| **Classes** | `normal/` (genuine), `tampered/` (forged) |
| **Resolution** | Variable → resized to 224×224 by the pipeline |
| **Download** | `git clone https://github.com/saifkhichi96/ssbi-dataset data/ssbi` then symlink/copy to `data/images/` |

**Why chosen:** Only publicly available real check image dataset with explicit forgery labels. Supplements the synthetic check images generated by `src/data/generate_synthetic.py`.

## 2. Evaluation Metrics

### 2.1 Primary Metrics (imbalance-aware)

| Metric | Rationale |
|---|---|
| **AUPRC** (Area Under Precision–Recall Curve) | Primary objective; robust to severe class imbalance |
| **AUROC** | Secondary; standard for ranking models |
| **F1 @ optimal threshold** | Balances precision and recall at the best operating point |

### 2.2 Explainability Metrics

| Metric | Method | Target |
|---|---|---|
| Attention entropy | Shannon entropy of modality weights | < 1.5 bits (focused attention) |
| Grad-CAM faithfulness | Mean drop in score after masking top-10% pixels | > 15% drop |
| Token attribution corr. | Pearson ρ between LIG weights and human-labelled suspicious words | ρ > 0.6 |

### 2.3 Ablation Study Design

Run four model variants to quantify each modality's marginal contribution:

| Run | Tabular | Image | Text | Expected AUPRC |
|---|---|---|---|---|
| `tab-only` | ✓ | — | — | Baseline |
| `tab+img` | ✓ | ✓ | — | +Δ₁ |
| `tab+txt` | ✓ | — | ✓ | +Δ₂ |
| `tab+img+txt` | ✓ | ✓ | ✓ | +Δ₃ (best) |

Hypothesis: multimodal AUPRC > any single-modality by ≥ 3 pp.

## 3. Training Protocol (Colab H100)

### 3.1 Recommended Configuration

```bash
# Single H100 — recommended for Colab Pro+
python -m src.training.train \
    --tabular-dir data/tabular \
    --paysim-path data/paysim/paysim.csv \
    --image-dir   data/images \
    --text-path   data/text/descriptions.csv \
    --batch-size  64 \
    --epochs      50 \
    --lr          1e-4 \
    --device      cuda
```

### 3.2 Multi-GPU (if A100 × 2 available)

```bash
torchrun --nproc_per_node=2 -m src.training.train --ddp --batch-size 32
```

### 3.3 Quick Dev Run (CPU / free Colab T4)

```bash
python -m src.training.train \
    --sample-size 2000 --device cuda --epochs 5 --batch-size 16
```

### 3.4 Hyperparameter Schedule

| Phase | Epochs | LR | Notes |
|---|---|---|---|
| Warmup | 1–3 | 1e-5 → 1e-4 | Linear ramp |
| Main | 4–40 | 1e-4 (CosineWarmRestart) | T₀=15, T_mult=2 |
| Fine-tune | 41–50 | 1e-5 | Unfreeze image + text layers |

## 4. Test Protocol

### 4.1 Hold-out Split

- **Train:** 70 % of IEEE-CIS + 70 % of PaySim, all SSBI train images
- **Val:** 10 % (early stopping on AUPRC)
- **Test:** 20 % (reported once, never used for tuning)

### 4.2 Robustness Tests

| Test | Method |
|---|---|
| Missing modality | Blank image (zeros) / empty text — score should degrade gracefully |
| High-value transaction | `TransactionAmt > $10,000` — check attention shift to tabular |
| Suspicious text | Known fraud phrases → text attribution weights should spike |
| Corrupted image | Gaussian noise added — verify attention falls on tabular |

### 4.3 Fairness Check

- Stratify test set by `DeviceType` (desktop vs. mobile) → report per-stratum F1
- Target: F1 variance < 5 pp across strata

---

## 5. Reporting Thresholds (Pass/Fail)

| Metric | Minimum Pass | Stretch Goal |
|---|---|---|
| AUPRC | ≥ 0.65 | ≥ 0.80 |
| AUROC | ≥ 0.90 | ≥ 0.95 |
| F1 (optimal threshold) | ≥ 0.55 | ≥ 0.70 |
| Multimodal vs. tabular-only AUPRC gain | ≥ 3 pp | ≥ 8 pp |
| Missing modality score degradation | < 10% drop | < 5% drop |
